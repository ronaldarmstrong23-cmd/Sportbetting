# NBA Playoff Research Betting Tool

This repository contains resources and tools for NBA playoff research and betting analysis during the 2026 playoffs.

## 📂 Project Structure

### Data Folders (`data/`)
- **odds/**: Betting odds from various sportsbooks and line movements
- **injuries/**: Player injury data and impact analysis
- **player-props/**: Player prop bets lines and statistics
- **box-scores/**: Game box scores and performance metrics

### Notes Folders (`notes/`)
- **game-breakdowns/**: Detailed game analysis and matchup breakdowns
- **parlay-builder/**: Parlay combination rules and strategies
- **injury-reports/**: Real-time injury impact reports

### Scripts Folder (`scripts/`)
- **stat-pulls/**: Scripts to fetch live player statistics and odds
- **simulations/**: Game outcome simulations and predictive models

### Outputs Folder (`outputs/`)
- **daily-picks/**: Daily betting picks and predictions
- **graded-picks/**: Historical performance tracking of past picks
- **screenshots/**: Visual evidence and documentation

## 🚀 Getting Started

1. Clone the repository
2. Navigate through folders to access data and scripts
3. Run simulation scripts to generate daily picks
4. Track your pick performance in graded-picks/
5. Update injury reports regularly before each game

## 📊 Key Files for Version Control

- `parlay-builder-rules.md` - Your parlay betting rules and strategies
- `injury-report-template.csv` - Template for tracking injuries
- `simulation-engine.py` - ML-based prediction engine
- `daily-picks-log.csv` - Historical log of all picks and grades

## 🎯 Betting Workflow

1. **Research Phase**: Analyze box-scores, odds, and injury reports
2. **Simulation Phase**: Run models to generate predictions
3. **Pick Generation**: Output daily picks based on simulations
4. **Tracking Phase**: Grade picks and maintain performance logs
5. **Analysis Phase**: Review trends and refine strategies

## ✅ Best Practices

- Always check injury reports before making picks
- Test strategies with simulations before wagering
- Keep detailed notes on reasoning for each pick
- Grade picks consistently for performance tracking
- Version control all strategy changes and rule updates
- Maintain accurate pick history for backtesting
